React Hook 学习笔记

上班以来学习React进行开发有一段时间了，由于是比较新的项目，当然也用上了React Hook，本萌新刚上手时一脸懵逼，不过好在React Hook是比较好理解的，在同事的一番点拨之下很快就上手学会了。
一般React Hook中用的最多的就是UseState和UseEffect，多用几次以后会发现非常好用

使用Hook的原则

  1. Hook的调用顺序不能改变，因为React依靠的是Hook的调用顺序来将state和其对应的Hook进行关联的。这意味着不能在循环、条件判断或者子函数中调用,要把Hook放在React函数最顶层以及任何的return之前，在每一次的渲染过程中，始终确保Hook的调用顺序不发生改变
  2. Hook本身是JS函数，但是Hook只能在React的函数组件中使用而不能在JS的函数中使用（当然也可以在Hook中使用Hook）

    useState

    useState用于声明一个变量和修改这个变量的方法，并给变量赋一个初值 这里我声明了一个变量a以及修改这个变量a的方法，同时给a赋初值0
    当然现在大型的项目一般都用Ts来写了，那就给上面的变量限定一个类型吧 当我们要修改变量时，只需要用到我们声明的那个修改变量的方法就行了
    比如现在要把变量a修改为3 值得注意的是，如果新的state需要通过原来的state计算出，那么应该穿给setState一个函数，函数接受原先的state作为参数，返回一个经过原来的state计算而出的新state

    useEffect

    useEffect是另一个常用的React Hook，用于声明一个在页面渲染时触发以及被依赖项的变化所触发的函数以及它的依赖项 useEffect接收两个参数，第一个参数是被处触发的函数，第二个参数是能触发函数执行的依赖项数组，我在声明时写了一个空数组，因为当前函数不需要依赖项。因此这个函数只会在页面刚渲染是触发一次，之后不会再被触发

自定义Hook

自定义Hook可以理解为一个自定义的函数，将一些可复用的逻辑放在一个自定义Hook中以调用Hook的方式去调用它。
需要注意的是:自定义Hook必须以”use”开头，这个约定非常重要，只有这样React才会自动检查你的Hook是否违反了上文提到的Hook规则。
